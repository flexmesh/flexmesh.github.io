FlexMesh

On-demand Feature Orchestration for P4-specific Programmable Data Plane


Find Out More View Source Code

Background of FlexMesh

P4 has started a new era of programming networks for its ability of defining data plane behaviors. By building P4 programs, network operators can introduce diverse novel features such as network telemetry, stateful packet filtering and novel routing protocols into network devices.

Motivations of FlexMesh

  • Complexity: Developing general-purpose P4 programs composed of various network features is error-prone and can be a source of complexity.
  • Inflexibility: The data plane supports a limited set of feature orchestrations and is incapable of on-demand feature orchestrations at runtime.

FlexMesh Offers

FlexMesh is proposed as a systematic framework P4.

  • Simplicity for building P4 programs
  • Flexibility for orchestrating network features
  • High Perfromance for enforcing network features

FlexMesh Framework

FlexMesh Core

The core code of FlexMesh framework. It supports for composition of features and defines the management componennts in FlexMesh, such as Rewinder and Initializer.

FlexMesh Modules

The feature library of FlexMesh. Operators can describe the orchestration of various network features by registering the required modules in the configuretion files.

Context of the modules. All components in the context and shared globally. And the context can beused to pass parameters among modules.
